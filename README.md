# CTF Training Lab

Учебный репозиторий для студенческого CTF-интенсива на **3 пары по 90 минут**. Задачи локальные, безопасные и рассчитаны на смешанную аудиторию: от новичков до студентов с опытом cybersecurity/CTF.

## Цели обучения

- Понять формат CTF и корректную сдачу флагов `edu_ctf{...}`.
- Освоить базовый порядок исследования файлов, программ, шифртекстов, контейнеров и учебных Web-сервисов.
- Научиться делать маленькие воспроизводимые задания и объяснять решение.
- Подготовиться к самостоятельному решению задач через CTFd.

## Категории

| Категория | Что тренируем |
|---|---|
| Reverse | Анализ программ, строк, простых проверок и байткода. |
| Forensics | Анализ артефактов, сигнатур, архивов, метаданных и PCAP. |
| Cryptography | Отличие encoding/hashing/encryption, Caesar, XOR, Vigenere, базовый RSA. |
| Steganography | Поиск скрытых сообщений в EXIF, LSB, аудио и контейнерах. |
| Web | Анализ HTTP-методов, заголовков, ввода, загрузки файлов и серверной обработки данных. |

## Структура трёх пар

1. **Теория и демонстрации**: преподаватель показывает 5 demo-задач, включая локальный Web-сервис, и даёт 4 файловые mini-задачи.
2. **Парный workshop**: один студент создаёт простой флаг-челлендж, второй решает, затем роли можно поменять.
3. **Финальный CTF**: самостоятельное решение базовой борды из 15 задач или расширенной борды из 20 final-задач через CTFd. Часть final-задач помечена как bonus/extra и может открываться сильным студентам или использоваться как запас.

## Структура репозитория

- `lessons/` — сценарии трёх пар.
- `cheatsheets/` — короткие шпаргалки по категориям и инструментам.
- `challenges/TEMPLATE/` — шаблон задачи для дальнейшего копирования.
- `challenges/intro/` — 13 intro-задач первой пары: demo, mini, дополнительные mini-задачи и Web-демонстрация.
- `challenges/final/` — 20 final-задач третьей пары, включая по четыре задачи в каждой из пяти категорий.
- `docker-compose.web.yml` — локальный запуск пяти изолированных Web-сервисов.
- `ctfd/` — заметки по переносу задач в CTFd.
- `docs/` — правила безопасности, модель баллов, чек-листы.
- `scripts/` — проверки флагов и упаковка публичных файлов.

## Как преподавателю использовать репозиторий

1. Прочитать `TEACHER_GUIDE.md` и `CHALLENGE_PLAN.md`.
2. Для каждой задачи скопировать `challenges/TEMPLATE/` в нужную папку.
3. Хранить исходники и генераторы в `src/`; публичные артефакты должны воспроизводимо создаваться в `dist/` генераторами.
4. Не коммитить сгенерированные файлы из `dist/`: в git остаётся только `dist/.gitkeep`, а реальные публичные файлы пересоздаются перед занятием.
5. Перед занятием запустить `bash scripts/build_all.sh`, чтобы сгенерировать все `dist/` и проверить флаги, solver-скрипты и структуру Web-сервисов.
6. Упаковать публичные файлы командой `python3 scripts/package_dist.py`; архивы из `build/packages/` можно загружать в CTFd.
7. Для Web-задач запустить `docker compose -f docker-compose.web.yml up --build -d` и `python3 scripts/smoke_web.py`.
8. Передать в CTFd описание, архивы/файлы из `dist/` либо URL Web-сервиса, подсказки и флаг.

## Как студентам получать dist-файлы

Студентам выдаются только публичные артефакты из `dist/`, zip-архивы из `build/packages/` и адреса запущенных Web-сервисов. Сгенерированные файлы из `dist/` не хранятся в git. Файлы `solve.md`, `flag.txt`, `src/`, Dockerfile и внутренние заметки преподавателя студентам не выдаются.

## CTFd

CTFd используется как борда для сдачи флагов и подсказок. Репозиторий не поднимает CTFd сам: коллега, отвечающий за инфраструктуру, переносит задачи по материалам из `ctfd/` и `challenge.yml`.

> Важно: `solve.md` и `flag.txt` нельзя загружать как публичные файлы в CTFd.

## Workflow разработки задач

1. Выбрать идею из `CHALLENGE_PLAN.md`.
2. Создать задачу из шаблона.
3. Для файловой задачи написать `src/generate.py`, создающий безопасные артефакты в `dist/`; для Web-задачи добавить Dockerfile и описание `service` в `challenge.yml`.
4. Описать решение, подсказки и флаг.
5. Проверить воспроизводимость через `bash scripts/build_all.sh`; Web-задачи дополнительно проверить через `scripts/smoke_web.py`.
6. Упаковать публичную часть файловых задач через `python3 scripts/package_dist.py`. Для Web-задачи передать в CTFd URL развернутого сервиса вместо архива.
