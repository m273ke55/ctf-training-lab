# Вторая пара: парный workshop

## Цель

Научить студентов создавать маленькие, объяснимые и решаемые CTF-задачи, а также решать задачи однокурсников.

## Правила работы в парах

- Автор должен сам объяснить решение своей задачи.
- Задача должна решаться за 5–15 минут.
- Флаг имеет формат `edu_ctf{...}`.
- Желательно оставить хотя бы одну подсказку.
- После решения роли можно поменять.

## Тайминг на 90 минут

| Время | Действие |
|---|---|
| 0–10 | Правила, безопасность, примеры хороших мини-задач. |
| 10–20 | Выбор пар и категорий. |
| 20–45 | Создание первой мини-задачи. |
| 45–60 | Решение первой мини-задачи партнёром. |
| 60–80 | Смена ролей или доработка. |
| 80–90 | Обсуждение и чек-лист качества. |

## Что делает автор

Описывает условие, создаёт безопасный артефакт или локальный Web-сервис, записывает флаг, готовит краткое решение и подсказку.

## Что делает решающий

Фиксирует гипотезы, задаёт уточняющие вопросы только после самостоятельной попытки и сообщает, где задача была непонятной.

## Запрещено

- Необратимый случайный процесс.
- Просто спрятать флаг без подсказок.
- Платные или редкие инструменты.
- Вредоносные файлы, персистентность и запросы к сервисам вне локального учебного стенда.
- Реальные персональные данные и опасные payload.

## Как обеспечить решаемость

Автор до обмена проходит задачу сам с нуля и записывает команды. Если решение нельзя объяснить за 2 минуты, задача слишком сложная.

## Оформление мини-задачи

Используйте карточку: название, категория, файлы или URL сервиса, флаг, шаги решения, подсказка, ожидаемое время.

## Допустимые идеи

| Категория | Идеи |
|---|---|
| Reverse | Флаг в строках, простая проверка ввода, Python-скрипт с обратимым преобразованием. |
| Forensics | Неверное расширение, метаданные, вложенный текстовый файл в архиве. |
| Cryptography | Caesar, ROT13, XOR с коротким ключом, Base64 как encoding. |
| Steganography | EXIF comment, текст после EOF, простая LSB-идея с готовым инструментом. |
| Web | Локальная HTML-страница, анализ заголовков, безопасная обработка параметра или различие HTTP-методов. |

## Шаблон карточки

```text
Название:
Категория:
Файлы или URL для решающего:
Флаг: edu_ctf{...}
Ожидаемое время: 5–15 минут
Идея решения:
Подсказка 1:
Запрещённые зависимости отсутствуют: да/нет
```

## Чек-лист автора

- [ ] Флаг в правильном формате.
- [ ] Есть решение и хотя бы одна подсказка.
- [ ] Нет вредоносного поведения.
- [ ] Нет редких инструментов.
- [ ] Задача решается за 5–15 минут.

## Чек-лист решающего

- [ ] Сначала проверены простые команды.
- [ ] Записаны гипотезы.
- [ ] Запрошена подсказка только после попытки.
- [ ] Найденный флаг сдан в формате `edu_ctf{...}`.

## Чек-лист преподавателя

- [ ] Пары сбалансированы.
- [ ] Слишком сложные идеи упрощены.
- [ ] Нерешаемые задачи остановлены до обмена.
- [ ] Безопасность проверена.
- [ ] Web-задача работает только на локальном учебном адресе.
