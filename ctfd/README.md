# Перенос задач в CTFd

Для каждой задачи перенесите название, категорию, описание, баллы, флаг, подсказки и публичные файлы из `dist/`. Файлы `solve.md` и `flag.txt` нельзя загружать как публичные: они раскрывают решение и флаг. `challenge.yml` можно использовать как внутреннюю карточку при ручном создании задачи.

Финальный workflow перед загрузкой в CTFd:

```bash
bash scripts/build_all.sh
python3 scripts/package_dist.py
```

После выполнения этих команд готовые zip-архивы лежат в `build/packages/`. Их можно использовать для передачи публичных материалов задач в CTFd.

Перед импортом пройдите финальный чек-лист: [`final_import_checklist.md`](final_import_checklist.md).
