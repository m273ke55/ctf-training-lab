# Шпаргалка по Web

## Базовый порядок

1. Откройте страницу и изучите URL, формы и ссылки.
2. Посмотрите запрос и ответ во вкладке Network браузера.
3. Проверьте HTTP-метод, статус, заголовки, параметры и cookies.
4. Повторите запрос через `curl`, меняя только один элемент за раз.
5. Работайте исключительно с адресом учебной задачи.

## Полезные команды

```bash
curl -i http://target/
curl -I http://target/admin
curl -X POST -d "name=value" http://target/
curl -F "file=@sample.txt" http://target/upload
```

## Типовые признаки

- Пользовательский ввод попадает в команду или интерпретатор.
- Имя файла проверяется только по расширению.
- Один маршрут по-разному обрабатывает HTTP-методы.
- XML-парсер разрешает DTD и внешние сущности.
- Сервер подключает локальный файл по пользовательскому параметру.

Web-уязвимость в учебной задаче является намеренной. Не переносите проверку на реальные сайты и сервисы.
